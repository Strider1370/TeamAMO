# 09. Spec (03 단계) — 가족 복지·돌봄 내비게이터

> 기준: [[08-concept]] (막막함 → 오늘 할 일, ★승인). 이 문서는 **4시간 빌드·5분 발표**용 확정 스펙.
> 원칙: 작은 앱(화면 2개) · 판정은 코드 · AI는 이해/번역 · 자격은 "받을 가능성+공식링크" · 모바일 우선.

## 1. Answer First
**어느 날 갑자기 부모님이 쓰러지면, 자녀는 누구나 처음입니다.** 받을 수 있는 도움은 분명 어딘가 있는데, 무엇부터·어디에 물어야 할지 몰라 검색창만 켜둔 채 막막한 그 밤 —
→ 상황을 그냥 **말하듯 풀어놓으면**, 흩어진 제도를 **「오늘 할 일」 순서**로 정리해 그 막막함의 **첫 줄을 대신 써주는** 작은 길잡이.
*(누구 = 처음 돌봄을 맡은 자녀 / 무엇 = 받을 가능성 있는 지원의 행동계획 / 어떻게 = 자연어 → AI 정리)*

## 1.1 톤 & 스토리 (발표 감성 throughline)
- **핵심 감정 곡선: 막막함 → 안심.** "혼자 검색창 앞에서 막막하던 밤" → "할 일이 정리된 아침."
- **오프닝 비트(약 10초, 구체적 한 장면):** 한 자녀가 부모 퇴원 통보를 받고 밤에 검색창만 켜둔 채 막막해하는 순간. 통계가 아니라 **사람 한 명의 한 순간**으로 연다.
- **감정 Job(리서치 트랙 A 근거):** "내가 뭔가 놓치고 있진 않을까"의 불안 → "놓친 게 없다는 안심", "혼자가 아니라는 느낌".
- **톤:** 따뜻하되 담백하게. 미사여구·과장(혁신적·게임체인저·seamless) 금지. 첫 문장은 사람 이야기, 마지막 문장은 막막함의 해소.
- **마지막 문장:** "처음이라 막막한 그 순간, 첫 줄을 대신 써드립니다."
- **규율:** 감정은 말로 호소하지 말고 **데모로 증명**(막막한 한 문단 입력 → 정리된 할 일 출력). 신파·동정 유발 과잉 ❌ — 심사위원은 피곤하다.

## 2. Problem (요약)
정보는 공개돼 있으나 **첫걸음의 부재**. 제도가 기관별로 흩어지고(건보공단·치매안심센터·복지로·지자체) 용어·순서를 모름. 3중 장벽 = 인지+번역+실행.

## 3. AI Leverage (역할 분담 = 정직성)
- AI 없이도: 키워드 카탈로그 검색.
- AI로 더 빠르게: 행정문어 → 쉬운 말 번역.
- **AI라서 처음 가능**: 두서없는 한 문장에서 **여러 대상·여러 제도 축 동시 해석** → 개인화 순서화.
- **AI가 하는 것**: 자연어→슬롯 추출, 제도 설명 번역, "왜 당신에게 해당될 수 있나" 1줄, (stretch) 후속질문 생성.
- **AI가 안 하는 것**: 자격 판정 ❌, 링크 생성 ❌(curated 화이트리스트만). 매칭·순서·중복배제는 **결정적 코드**.
- **LLM**: OpenAI **GPT-5.5**(텍스트+이미지 멀티모달 — 자연어·카메라 서류 한 모델). 키=런타임 입력(⚙️), env, 깃 ❌. 라이브 지연 시 mini 폴백.
- **폴백**: LLM 키 없으면 slot-dictionary 키워드 매칭만으로 동작(데모 안전망).

## 4. Differentiation (킥)
"받을 수 있는 것의 목록"이 아니라 **막막한 사람을 위한 첫걸음·순서**. 화면에 보이는 3요소:
1. **의존성** — "① 장기요양 등급 → 그래야 ② 복지용구가 열림"
2. **중복배제** — "장기요양 ↔ 노인맞춤돌봄 택일"
3. **시간축** — 지금 / 퇴원 전·후 / 장기

## 5. Solution — 입력 → 처리 → 출력 → 다음 행동
> 레이아웃 기준(블루프린트): `design/mockups/01-input-result.html`(입력·이해중·타임라인 결과) · `design/mockups/02-questionnaire.html`(질문지 보조입구). 빌드는 이 목업을 KRDS/Next로 재구현.

### 5.1 입력 (화면 1)
- **[P0] 자연어 한 칸** (큰 textarea): "부모님 상황을 편하게 적어주세요." 주인공.
- **[P0] 예시 칩 3개** (탭하면 채워짐) — demo-personas에서.
- **[P0] 지역 선택** (시도→시군구, 선택사항) — 지자체 제도 필터.
- **[P1] 질문지 보조 입구**: "글로 쓰기 막막하면 몇 개만 골라보세요" → "어떤 일이 생겼나요?" 10카드 + 후속 2~3문항 → **자연어와 같은 슬롯**으로 합류(questionnaire.json).
- **[P0] 설정 ⚙️**: LLM 키 런타임 입력 모달(메모리만, 저장 안 함). 없으면 폴백 안내.
- CTA: **"정리해줘"**.

### 5.2 처리 (전환 상태 — 신뢰/Wow)
- "상황을 이해하는 중…" + **추출된 슬롯 칩 노출**: `어머니: 치매 초기·거동 불편` `독거` `경기도`. AI가 읽은 것을 보여줌.
- 파이프라인: 슬롯 추출(AI/폴백) → 규칙 매칭(situations 점등) → 후보 제도 수집 → **위상정렬(prerequisiteOf)로 순서** → exclusiveWith 가지치기 → 타임라인 배치 → 링크·배지·서류는 데이터.

### 5.3 출력 (화면 2 — 오늘 할 일)
- 헤더 요약: "어머니(치매 초기·독거)에 대해 **N개의 할 일**을 찾았어요."
- **「지금 / 퇴원 전·후 / 장기」 타임라인 섹션**. 각 섹션에 카드 리스트.
- **카드 1장 필드**:
  - 제도명 + **배지**(`받을 가능성` / `확인 필요`)
  - *왜 해당될 수 있나* (AI 번역 1줄)
  - 추가 확인 조건
  - 신청처·방법 (온/방/전 + 전화번호)
  - 준비 서류
  - **공식 링크 버튼**
  - 의존성 뱃지("먼저: 장기요양 등급" / "이게 되면 열림")
  - 중복배제 뱃지("택일: 노인맞춤돌봄")
- **[P2] 하단**: "이 외에 더 있을 수 있어요 — 공식 복지 검색"(광역 16,000 레이어) · 가족 공유(PDF/링크).

### 5.4 다음 행동
순서대로 신청처 연락/방문. 각 카드의 공식 링크·전화로 바로 연결.

## 6. 데이터
- **[P0] `data/welfare/curated-benefits.json`** (~80제도, 정밀 레이어) — Wow 엔진.
- **[P0] `data/questionnaire/`** (questionnaire.json·slot-dictionary.json) — 입력→슬롯.
- **[P0] `data/personas/demo-personas.json`** — 예시 입력·테스트 픽스처(가상, 엔진이 실제 처리).
- **[P2] gov24·복지 16,000 스냅샷** — 광역 그물(번들 확보 시).

## 7. Demonstration (5분 한 장면)
페르소나 ①(뇌졸중 퇴원+반신마비+독거 어머니+형편 빠듯) 입력 →
처리 화면에 슬롯 노출 → 결과: **「지금」** 장기요양 등급신청·재난적의료비(180일) / **「퇴원 후」** 복지용구·방문요양(등급 선행 표시) / **「장기」** 장애등록·자녀 가족돌봄휴직. 중복배제·공식링크·"확인 필요" 배지까지. → "막막한 한 문단이 순서 있는 할 일로." Wow는 **화면에서 검증 가능**.

## 8. 범위 (4시간)
- **P0 (필수, 데모 핵심경로)**: 입력(자연어+지역+예시칩) · 슬롯추출(AI+폴백) · 규칙매칭+순서화+중복배제 · 타임라인 결과 카드 · **전화연결(tel:)** · KRDS 모바일 · 키 모달.
- **P1 (코어 후, 이 순서)**: 질문지 보조 입구 · **GPS 지역자동** · **카카오 가까운 기관 지도/길찾기** · **카메라 서류입력(GPT-5.5 비전)** · **이미지 저장/공유**.
- **P2 (stretch)**: 광역 16,000 검색 · **푸시 알림 껍데기(시연용·정직 프레이밍)** · AI 후속질문 · 아이 도메인 토글.
- **제외(이번 X)**: 음성 입력(확장축) · 로그인/계정.
- 외부 의존성: OpenAI(LLM) · 카카오(지도/검색) — 키는 런타임/env, 전날 미리 발급. (상세: [[11-native-mobile]])
- 컷 규칙: 시간 부족 = 단계 생략❌ → **화면 수/기능 범위 축소**⭕ (데이터·핵심경로는 유지). 못 넣는 P1 = 확장 슬라이드.

## 9. Guardrail (정직성)
- 자격 단정 ❌ → "받을 가능성 + 공식 링크 확인". 핵심 경로는 AI 자유분류 ❌, 규칙 고정.
- 링크는 curated 화이트리스트만(환각 0). 데모 입력은 가상 페르소나. 민감정보 외부 LLM 전송 ❌.
- 의료·법률·금전은 안내까지, 최종 판단·신청은 사람.

## 10. 확장 (발표 마지막 1장)
같은 엔진으로 아동 도메인 커버 — "처음 겪는 가족 사건 → 행동계획" 플랫폼([[07-child-domain]]). MVP는 부모.

## 11. 확정된 결정
- LLM = **OpenAI GPT-5.5**(비전 포함, 런타임 키) · 지도 = **카카오**(JS+REST 키, 사용자 발급) · 질문지 = 병용 보조입구(자연어가 얼굴)
- 네이티브: 전화=P0 / GPS·지도·카메라·이미지공유=P1 / 푸시껍데기=P2 / 음성=제외 · 아이 도메인=확장 슬라이드

## 12. 검토 P0 구멍 해결 (10-spec-review 반영 — 빌드 첫 작업)
1. **규칙엔진 + 슬롯 어휘 통일** ⭐ — `data/welfare/rules.json` 생성: `슬롯/키워드 → situation 점등 → 제도`. 슬롯 어휘는 **questionnaire.json `slotSchema` 기준 1개로 통일**(persona·slot-dictionary·curated 정렬). 이게 없으면 매칭 0건 → **빌드 task 1**.
2. **timeline 교정** — 데이터 `timeline` 재분류: 급한 관문=`지금` / 등급의존 서비스=`퇴원후` / 별도트랙·돌봄자·예방=`장기`. (장애등록·가족돌봄휴직 → `장기`)
3. **confidence → 배지** — `확인`·`잠재후보`→"받을 가능성" / `미확인`→"확인 필요". (12건 미확인이 안 새게)
4. **LLM 배선** — `lib/llm.ts` 모델 `gpt-4.1-mini`→`gpt-5.5`, 비전 입력 경로 추가, 키 모달 카피.
5. **데이터 경로** — API 라우트가 `process.cwd()→../data/welfare/curated-benefits.json` 를 서버사이드 `fs`로 읽기(INDEX 패턴). curated/rules/questionnaire 동일.
- 🟡 나중: 빈결과·null링크·LLM실패 상태 / 카드에 `deadline.note`(대기공백)·`regionVaries` 노출 / "복지로 6단계 vs 1문장" 방어 카피.
